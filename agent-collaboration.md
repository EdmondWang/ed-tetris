# 抖音小游戏-俄罗斯方块开发 Agent 协作方案

## 1. Agent 职责分工

| Agent 类型 | 核心职责 | 具体任务 |
|------------|----------|----------|
| **资深前端工程师** | 游戏核心开发 | 1. 重构游戏架构，实现俄罗斯方块核心逻辑<br>2. 开发游戏 UI 界面和交互控制<br>3. 实现游戏状态管理和计分系统<br>4. 优化游戏性能和用户体验 |
| **游戏美工设计师** | 视觉效果设计 | 1. 设计俄罗斯方块的视觉风格和主题<br>2. 优化游戏界面的视觉效果和动画<br>3. 设计控制按钮和 UI 元素的样式<br>4. 确保视觉风格统一，符合太空主题 |
| **游戏测试工程师** | 测试与质量保障 | 1. 设计游戏测试用例<br>2. 执行功能测试，验证游戏逻辑正确性<br>3. 进行性能测试，确保流畅运行<br>4. 测试不同设备的兼容性<br>5. 提供测试报告和优化建议 |
| **搜索 Agent** | 技术调研与资料收集 | 1. 查找抖音小游戏开发最佳实践<br>2. 收集俄罗斯方块游戏的经典实现方案<br>3. 搜索相关技术文档和示例代码<br>4. 提供技术选型建议 |
| **资深后端工程师** | 可选支持（如需） | 1. 开发游戏排行榜功能（可选）<br>2. 实现游戏数据统计和分析（可选）<br>3. 提供后端 API 支持（可选） |

## 2. 协作流程

### 阶段一：规划与准备（1-2天）
- **搜索 Agent**：调研抖音小游戏开发规范和俄罗斯方块最佳实践
- **资深前端工程师**：分析现有代码架构，制定重构方案
- **游戏美工设计师**：设计游戏视觉风格和主题方案

### 阶段二：核心开发（3-4天）
- **资深前端工程师**：
  1. 重构 game.js 文件，替换游戏配置常量
  2. 实现俄罗斯方块核心逻辑（形状定义、移动、旋转、碰撞检测）
  3. 开发游戏 UI 界面和交互控制
  4. 集成音效系统

### 阶段三：视觉优化（2-3天）
- **游戏美工设计师**：
  1. 为俄罗斯方块设计视觉效果和动画
  2. 优化游戏界面的视觉表现
  3. 设计控制按钮和 UI 元素样式
- **资深前端工程师**：实现美工设计的视觉效果

### 阶段四：测试与优化（2-3天）
- **游戏测试工程师**：
  1. 执行功能测试，验证游戏逻辑
  2. 进行性能测试，确保流畅运行
  3. 测试不同设备兼容性
  4. 提交测试报告和问题清单
- **资深前端工程师**：根据测试报告修复问题，优化性能
- **游戏美工设计师**：根据测试反馈优化视觉效果

### 阶段五：发布准备（1天）
- **资深前端工程师**：
  1. 优化代码，压缩资源
  2. 更新游戏配置文件
  3. 准备发布所需的资源和文档
- **游戏测试工程师**：进行最终测试，确保游戏符合发布标准

## 3. 协作要点

### 3.1 沟通与同步
- 定期同步开发进度和遇到的问题
- 保持设计风格和技术实现的一致性
- 及时反馈和调整开发方案

### 3.2 代码管理
- 遵循统一的代码规范和命名约定
- 定期提交代码，确保代码的可维护性
- 做好代码注释，方便团队协作

### 3.3 质量保障
- 每个功能模块开发完成后进行单元测试
- 定期进行集成测试，确保模块间的兼容性
- 重视测试反馈，及时修复问题

### 3.4 资源管理
- 统一管理游戏资源文件（图片、音频等）
- 确保资源大小符合抖音小游戏的要求
- 优化资源加载性能

## 6. 协作注意事项

1. **明确任务边界**：每个 Agent 专注于自己的核心职责，避免职责重叠
2. **保持目标一致**：所有 Agent 围绕同一个项目目标协作
3. **及时沟通反馈**：遇到问题及时沟通，确保项目顺利进行
4. **遵循开发规范**：统一遵循抖音小游戏的开发规范和最佳实践
5. **重视用户体验**：所有设计和开发都以用户体验为中心

通过合理的分工和协作，可以充分发挥各个 Agent 的优势，高效完成俄罗斯方块游戏的开发。
